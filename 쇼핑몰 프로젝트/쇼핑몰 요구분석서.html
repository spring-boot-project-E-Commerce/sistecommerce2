1.1 상품 목록 조회

요구사항

  • 사용자는 등록된 상품 목록을 조회할 수 있어야 한다.
  • 상품 목록에는 상품명, 대표 이미지, 가격, 판매상태, 별점, 리뷰 수가 표시되어야 한다.
  • 상품 목록은 판매량순을 기본 정렬 기준으로 하되 가격, 최신, 별점 순서로도 조회할 수 있어야 한다.
  • 상품 목록은 가격, 별점, 핫딜여부로 필터링이 가능해야 한다.
  • 상품이 많을 경우 페이징 방식으로 조회할 수 있어야 한다.
  • 상품 목록은 카테고리 별로 필터링이 가능해야 한다.
  • 단종(판매중지)된 상품은 숨김 처리를 한 후 일반 사용자에게 노출되지 않아야 한다.
  • 품절 상품은 목록에 표시할 수 있지만, 구매는 불가능해야 한다.
  • 사용자는 상품명을 기준으로 상품을 검색할 수 있어야 한다.
  • 사용자는 최근 검색한 검색어를 확인할 수 있어야 한다.
  • 입력한 검색어가 상품명에 포함된 상품을 조회할 수 있어야 한다.
  • 대소문자, 띄어쓰기 차이에 영향을 최소화하여 검색할 수 있어야 한다.
  • 검색 결과는 상품 목록 형태로 제공되어야 한다.
  • 최근 조회한 상품을 목록 형태로 제공해야 한다.

1.2 상품 상세 조회

요구사항

  • 사용자는 상품 목록에서 특정 상품을 선택하여 상세 정보를 조회할 수 있어야 한다.
  • 상품 상세 화면에는 상품명, 가격, 이미지, 설명, 재고, 판매상태(품절), 판매처 정보가 표시되어야 한다.
  • 사용자는 상품 상세 화면에서 상품 옵션 선택이 가능해야 한다. (색상, 사이즈 등)
  • 상품 상세 화면에는 장바구니 담기, 바로구매 버튼이 제공되어야 한다.
  • 품절 상품은 장바구니 담기와 바로구매가 제한되어야 한다.
  • 상품 상세 화면에서 리뷰 목록과 평균 별점을 확인할 수 있어야 한다.
  • 상품 상세 화면에서 해당 상품의 가격분석 정보를 확인할 수 있어야 한다.
  • 상품이 단종된 경우 일반 사용자는 접근할 수 없어야 한다.
  • 일반 사용자는 상품에 대한 문의가 가능해야 한다.
  • 사용자가 선택한 상품과 유사한 상품을 확인할 수 있어야 한다.
  • 사용자가 상세조회를 할 경우 해당 제품의 조회수를 추가하고 저장해야 한다.

  • 찜 기능 추가 고려
  • 재입고 알림 기능 추가 고려

재고 표시재고가 0개이면 품절로 표시한다.
판매상태판매중, 품절, 판매중지 상태로 구분한다.
평균 별점1.0점부터 5.0점까지 소수점 첫째 자리로 표시한다.
조회수상품 상세 조회 시 1회 증가한다.

1.3 상품 CRUD 기능

요구사항

  • 판매처는 상품을 등록할 수 있어야 한다.
  • 판매처는 자신이 등록한 상품을 수정할 수 있어야 한다.
  • 판매처는 자신이 등록한 상품을 취소할 수 있어야 한다.
  • 의류 C,R,U 기능은 사이즈 선택이 가능해야 한다.
  • 상품은 검색, 정렬, 필터링이 가능해야 한다.
  • 관리자는 부적절한 상품을 숨김 처리할 수 있어야 한다.

상품 이미지최소 1개 이상 등록해야 한다.

1.4 장바구니 기능

요구사항

  • 회원은 원하는 상품을 장바구니에 담을 수 있어야 한다.
  • 회원은 장바구니 상품 수량을 변경할 수 있어야 한다.
  • 회원은 장바구니 상품을 삭제할 수 있어야 한다.
  • 회원은 장바구니에서 선택한 상품만 주문할 수 있어야 한다.
  • 품절되거나 삭제된 상품은 주문할 수 없어야 한다.
  • 장바구니에 담긴 상품의 총 금액을 확인할 수 있어야 한다.

상세 조건

항목조건
장바구니 담기판매중 상태의 상품만 장바구니에 담을 수 있다.
중복 상품 처리이미 장바구니에 담긴 상품을 다시 담으면 기존 수량에 추가하거나 안내 메시지를 표시한다.
상품 수량최소 1개 이상 선택해야 한다.
최대 수량상품 재고 수량을 초과할 수 없다.
수량 변경회원은 장바구니 화면에서 상품 수량을 증가 또는 감소시킬 수 있다.
상품 삭제회원은 장바구니에 담긴 상품을 개별 삭제할 수 있다.
선택 삭제회원은 여러 상품을 선택하여 한 번에 삭제할 수 있다.
전체 삭제회원은 장바구니 전체 상품을 삭제할 수 있다.
선택 주문회원은 장바구니 상품 중 선택한 상품만 주문할 수 있다.
총 금액 계산선택한 상품의 가격과 수량을 기준으로 총 금액을 계산한다.
품절 상품 처리품절 상품은 장바구니에는 표시되지만 주문할 수 없다.
판매중지 상품 처리판매중지 상품은 주문할 수 없으며 주문 대상에서 제외한다.

주요 데이터

  • 장바구니번호
  • 회원번호
  • 상품번호
  • 수량
  • 추가일

1.5 주문 기능

요구사항

  • 회원은 상품을 주문할 수 있어야 한다.
  • 주문 시 배송지 정보를 입력 또는 선택 할 수 있어야 한다.
  • 주문 완료 후 주문 내역이 생성되어야 한다.
  • 회원은 마이페이지에서 주문 내역을 확인할 수 있어야 한다.
  • 주문은 결제 상태, 배송 상태, 취소/반품 상태를 함께 관리해야 한다.
  • 상품 재고는 주문 완료 또는 결제 완료 시 차감되어야 한다.

주요 데이터

  • 회원번호
  • 주문일
  • 주문상태
  • 총 주문금액
  • 배송지
  • 수령인
  • 연락처

1.6 결제시스템

요구사항

  • 회원은 주문 상품에 대해 결제를 진행할 수 있어야 한다.
  • 결제수단은 API를 통해 해당하는 수단으로 결제 한다.
  • 결제 성공 시 주문 상태가 결제완료로 변경되어야 한다.
  • 결제 실패 시 사용자에게 실패 메시지를 보여줘야 한다.
  • 결제 내역은 주문 내역과 연결되어야 한다.
  • 결제 취소 시 결제 상태가 취소로 변경되어야 한다.
  • 상품 결제에 핫딜과 쿠폰을 적용할 수 있어야 한다.
  • 쿠폰 사용에는 특정 조건이 포함될 수 있다.
  • 결제 내역에는 구매한 상품, 원가, 핫딜 적용 할인가, 쿠폰 적용 할인가, 최종 구매가가 기록되어야 한다.

상세 조건

항목조건
주문 가능 사용자로그인한 회원만 주문할 수 있다.

  • 회원은 주문한 상품의 배송 상태를 확인할 수 있어야 한다.
  • 배송 상태는 단계별(시작전, 배송중, 도착완료)로 변경되어야 한다.

1.13 쿠폰 기능

  • 쿠폰에는 최소 주문 금액이 존재해야 한다.

2. 비기능 요구사항

2.1 보안

  • 회원 비밀번호는 암호화하여 저장해야 한다.
  • 로그인한 회원만 장바구니, 주문, 결제, 리뷰 기능을 사용할 수 있어야 한다.
  • 본인의 주문, 장바구니, 리뷰만 수정/삭제할 수 있어야 한다.
  • 관리자 권한이 필요한 기능은 일반회원이 접근할 수 없어야 한다.

2.2 데이터 무결성

  • 주문이 완료되면 상품 재고가 정확히 차감되어야 한다.
  • 결제 실패 시 주문 상태가 결제완료로 변경되면 안 된다.
  • 주문취소 또는 반품 완료 시 재고와 결제 상태가 함께 처리되어야 한다.
  • 삭제된 상품은 주문할 수 없어야 한다.

3. 적립금

  • 여유가 되면 구현하기로 한다.
